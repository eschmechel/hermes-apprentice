# Operations

Day-to-day operations guide for running an Apprentice deployment.

## Cron schedule

Register these with your scheduler (crontab, systemd timers, etc.):

```cron
# Every minute: deliver queued messages and check for operator replies
* * * * * apprentice-telegram dispatch-one
* * * * * apprentice-telegram poll-replies

# Every 2 minutes: process approved training jobs
*/2 * * * * apprentice-orchestrator tick

# Every 10 minutes: evaluate canary shadow data
*/10 * * * * apprentice-orchestrator safety advance --all
```

The installer prints the exact lines during `apprentice-setup --apply`.

## Starting services

### Option 1: Docker Compose

```bash
docker compose -f deploy/docker/docker-compose.yml up -d
```

Starts: proxy, serve (vLLM), observer, hermes, registry, burst.

### Option 2: Manual

```bash
observer serve --listen :8081 --hermes-db ~/.hermes/state.db &
proxy serve --listen :8083 --upstream-url https://openrouter.ai/api/v1 &
apprentice-serve --port 8000 --enable-lora &
```

### Option 3: systemd

```bash
# Per-pattern vLLM (template unit)
systemctl enable apprentice-vllm@<pattern-id>.service --now
```

Template unit at `serving/contrib/apprentice-vllm@.service`.

## Common tasks

### Check canary status

```bash
curl http://localhost:8083/canary/state | jq '.[] | select(.state=="broken")'
```

### Manually demote a broken specialist

```bash
curl -X POST http://localhost:8083/canary/set-state \
    -d '{"pattern_id":"p1","state":"broken","pct":0}'
```

### Check budget

```bash
apprentice-orchestrator budget get --tenant acme
```

### Increase budget (after Telegram approval)

```bash
apprentice-orchestrator budget increase --tenant acme --amount 50
```

### View GPU status

```bash
nvidia-smi
apprentice-orchestrator burst check --gpu A100
apprentice-orchestrator burst list-pods
```

### List tenants

```bash
apprentice-orchestrator quota list
```

### Rotate a tenant API key

```bash
head -c 32 /dev/urandom | base64 > ~/.apprentice/tenants/<tenant>/.apikey
# Restart proxy to pick up new key
```

## Upgrading

### v0.1 → v0.2

See [Migration Guide](../migration-v0.1-to-v0.2.md) for full details. Key changes:
- Registry format now includes `base_model` and optional `merged_from`/`aliases` fields.
- Pattern store now has `tenant_id` field — existing patterns default to global (no tenant).
- Canary ramp state persisted separately — proxy restart no longer drops ramp positions.
- Installer replaces manual loop-based setup.

### Docker image updates

```bash
docker compose -f deploy/docker/docker-compose.yml build --no-cache
docker compose -f deploy/docker/docker-compose.yml up -d
```

## Backup

Critical paths to back up:

```bash
~/.apprentice/.env                  # Environment config
~/.apprentice/keys/                 # Ed25519 signing keys
~/.apprentice/tenants/              # Tenant API keys + quotas + budget
~/.apprentice/registry/             # Promoted specialist models
~/.apprentice/proxy/patterns.json   # Pattern centroids
~/.apprentice/proxy/canary/         # Canary ramp state
```

Registry models are the heaviest — they can be regenerated by re-running training if disk space is a concern.

## Troubleshooting

**Proxy returns 502 for all requests:**
- Check `--upstream-url` is correct and reachable.
- Check `OPENROUTER_API_KEY` is set.
- Verify no firewall blocking outbound HTTPS.

**Specialist never promoted (validation fails):**
- Check `~/.apprentice/failures/` for the failure report.
- Compare specialist scores vs baseline in the report.
- Ensure test dataset has enough samples (minimum ~20).

**Canary stuck at low percentage:**
- Check shadow agreement scores: `GET /canary/state/<id>`.
- Agreement below 0.80 means specialist responses differ significantly from upstream.
- Consider re-training or checking for prompt mismatch.

**Multi-LoRA residency errors:**
- Check `GET /residency/status` for resident adapter list.
- Ensure vLLM was started with `--enable-lora --max-loras`.
- Check GPU memory: `nvidia-smi`.

**Telegram messages not delivered:**
- Check outbox: `apprentice-telegram list`.
- Verify Hermes cron job is running: `hermes cron list`.
- Check Telegram bot token has message permissions.
